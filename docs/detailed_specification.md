# [상세 명세서] 블로그 올인원 운영 시스템 (Handover to Codex)

## 0. 핵심 정책 (Policy Fix)
- **1차 MVP 범위**: 주제 추천 / 포스팅 제목·본문 초안 / 해시태그 / 기본 대시보드
- **댓글/대댓글 자동화**: 2단계 확장 기능으로 분리 (1차 MVP 본체 제외)
- **저장 정책**: 1차 MVP는 브라우저 로컬 스토리지 기반의 **임시 저장** 우선
- **상품 전략**: 완전 자동화가 아닌 사용자가 검수하는 **관리형/반자동형** 솔루션
- **핵심 화면**: 콘텐츠 메이커 (Maker) 화면이 1차 개발의 핵심

---

## 1. 화면별 요구사항 명세

### 1) 대시보드 (Dashboard)
- **목적**: 시스템 사용 현황 파악 및 최근 작업물로의 빠른 접근
- **사용자 입력값**: 없음 (조회 전용)
- **출력값**: 누적 생성 글 수, 오늘 작업 횟수, 최근 작성 초안 리스트 (최대 5개)
- **버튼 동작**: 
    - [새 글 쓰기]: 콘텐츠 메이커 화면으로 이동
    - [초안 클릭]: 해당 초안을 콘텐츠 메이커 결과창에 로드하여 이동
- **완료 조건**: 대시보드 진입 시 로컬 스토리지 데이터를 읽어와 정확한 통계와 리스트가 표시됨

### 2) 콘텐츠 메이커 (Content Maker) - **핵심 화면**
- **목적**: AI를 활용한 블로그 포스팅 반자동 생성
- **사용자 입력값**: 키워드, 업종/주제, 글 목적(정보형/후기형/홍보형), 말투, 이모지 사용 여부(Y/N), 금지어
- **출력값**: 추천 주제 3개, 제목 후보 3개, 본문 초안 1개, 해시태그 10개
- **버튼 동작**:
    - [생성]: 입력값을 바탕으로 주제/제목/본문/태그 생성 시작
    - [다시 생성]: 현재 입력값 유지한 채 새로운 결과 도출
    - [수정]: 생성된 결과창(에디터)에서 텍스트 직접 편집 가능
    - [복사]: 제목, 본문, 태그를 각각 클립보드에 복사
- **완료 조건**: 생성 프로세스 완료 후 결과창에 모든 출력값이 정상 표시되고, 사용자가 수정한 내용이 실시간 반영됨

### 3) 내 보관함 (My Storage)
- **목적**: 작성된 초안들의 로컬 아카이빙 및 관리
- **사용자 입력값**: 검색어 (제목 기반)
- **출력값**: 저장된 초안 리스트 (제목, 날짜, 키워드 요약)
- **버튼 동작**:
    - [열기]: 해당 초안을 콘텐츠 메이커로 불러오기
    - [삭제]: 로컬 스토리지에서 해당 데이터 영구 삭제
- **완료 조건**: 로컬 스토리지에 저장된 JSON 데이터를 파싱하여 리스트를 정확히 렌더링함

### 4) 고급 설정 (Settings)
- **목적**: 시스템 환경 설정 및 확장성 확보
- **사용자 입력값**: Gemini API Key (관리자용), 블로그 기본 정보
- **출력값**: API 연결 상태, 현재 버전 정보
- **버튼 동작**:
    - [저장]: 설정값 로컬 저장
    - [연결 테스트]: API 키 유효성 검사
    - [댓글 자동화(슬롯)]: "준비 중" 메시지 표시 (1차 MVP 미구현)
- **완료 조건**: 설정 저장 시 시스템 전역에서 해당 API 키를 사용하여 통신 가능함

## 2. 콘텐츠 메이커 상세 명세 (MVP 핵심)

- **입력 필드 상세**:
    - `Keyword`: 메인 키워드 (Input Text)
    - `Category`: 업종/주제 (Select Box)
    - `Goal`: 글 목적 (Radio: 정보형, 후기형, 홍보형)
    - `Tone`: 말투 (Select: 친절한, 전문적인, 유머러스한 등)
    - `Emoji`: 이모지 사용 여부 (Toggle/Switch)
    - `Avoid`: 금지어 (Input Text, 쉼표 구분)
- **버튼 흐름**:
    - `생성` → `상태: 생성 중` → `결과 표시` → `상태: 생성 완료`
    - 결과 확인 후 사용자가 에디터 클릭 시 → `상태: 수정 중`
    - `복사` 버튼 클릭 시 → `상태: 복사 완료` 알림 토스트

---

## 3. 상태 흐름 정의 (State Transitions)

1. **[입력 전]**: 입력 필드가 비어있고 '생성' 버튼 비활성화
2. **[입력 완료]**: 필수 입력값 충족 시 '생성' 버튼 활성화
3. **[생성 중]**: 로딩 애니메이션 표시, 버튼 비활성화
4. **[생성 완료]**: 결과창에 텍스트 렌더링, '복사/저장' 버튼 노출
5. **[수정 중]**: 결과창 에디터 포커스 시 상태 변경, 변경 사항 자동 저장 준비
6. **[저장됨]**: 사용자가 명시적으로 저장하거나 자동 저장 완료 시 로컬 데이터 업데이트
7. **[복사 완료]**: 클립보드 복사 성공 시 성공 토스트 메시지 출력

## 4. 코덱스(Codex) 인계용 체크리스트

### ❶ 기술 구조 및 환경
- [ ] 폴더 구조: `/frontend`, `/backend`, `/shared` 기반 구성
- [ ] 메뉴 구조: 대시보드 / 콘텐츠 메이커 / 내 보관함 / 고급 설정
- [ ] 상태 관리: `Zustand` 또는 `Context API`를 활용한 로컬 데이터 전역 관리
- [ ] 라우팅: `react-router-dom`을 이용한 사이드바 기반 네비게이션

### ❷ 화면별 입출력 및 동작
- [ ] **콘텐츠 메이커**: 키워드, 업종, 목적, 말투, 이모지, 금지어 입력 처리
- [ ] **콘텐츠 메이커**: 주제(3), 제목(3), 본문(1), 태그(10) AI 생성 및 출력
- [ ] **버튼 동작**: 생성, 다시 생성, 수정, 복사 기능 정상 작동 여부

### ❸ 개발 우선순위 (1차)
1. Vite + React + Tailwind + Express 프로젝트 기본 환경 구축
2. 콘텐츠 메이커의 입력 폼 및 결과창 에디터 UI 구현
3. Gemini API 연동 (주제 추천 → 제목/본문 생성 로직)
4. 로컬 스토리지를 활용한 초안 저장 및 보관함 리스트 연동

### ❹ Acceptance Criteria (수락 기준)
- [ ] 모든 메뉴가 정상적으로 라우팅되는가?
- [ ] AI 생성 결과가 에디터에서 즉시 수정 가능한가?
- [ ] 제목/본문/태그가 각각 원클릭으로 클립보드에 복사되는가?
- [ ] 새로고침 후에도 작성 중인 데이터가 로컬에 유지되는가?
