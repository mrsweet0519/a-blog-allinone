# [상세 명세서] 블로그 올인원 운영 시스템 (Handover to Codex)

## 0. 핵심 정책 (Policy Fix)
- **1차 MVP 범위**: 주제 추천 / 포스팅 제목·본문 초안 / 해시태그 / 기본 대시보드
- **베타 테스트 주요 기능**: 콘텐츠 메이커, 캡처 이미지 기반 댓글 추출, 수동 댓글 입력 기반 대댓글 생성, 복사 흐름 중심
- **정적 베타 유지 기능**: 콘텐츠 메이커, SEO/AEO 점검, 접속코드 인증, 수동 댓글 입력, 캡처 업로드, OCR 원문/댓글 카드 분리, 대댓글 생성/복사 기능은 정적 베타 모드에서도 유지
- **판매용 접속 제어**: 1차 판매 단계에서는 정식 회원가입 대신 접속코드와 만료일 기반 게이트를 사용하고, 추후 Supabase 또는 Cloudflare Workers/KV 검증으로 교체 가능하게 분리
- **댓글/대댓글 자동화**: URL 자동 댓글 수집/자동 등록은 2단계 확장 기능인 로컬 `blog-automation` 브리지 기능으로 분리
- **저장 정책**: 1차 MVP는 브라우저 로컬 스토리지 기반의 **임시 저장** 우선
- **상품 전략**: 완전 자동화가 아닌 사용자가 검수하는 **관리형/반자동형** 솔루션
- **핵심 화면**: 콘텐츠 메이커 (Maker) 화면이 1차 개발의 핵심

## 1. 화면별 요구사항 명세

### 1) 대시보드 (Dashboard)
- **목적**: 시스템 사용 현황 파악 및 최근 작업물로의 빠른 접근
- **사용자 입력값**: 없음 (조회 전용)
- **출력값**: 누적 생성 글 수, 오늘 작업 횟수, 최근 작성 초안 리스트 (최대 5개)
- **버튼 동작**: 
    - [새 글 쓰기]: 콘텐츠 메이커 화면으로 이동
    - [초안 클릭]: 해당 초안을 콘텐츠 메이커 결과창에 로드하여 이동
- **완료 조건**: 대시보드 진입 시 로컬 스토리지 데이터를 읽어와 정확한 통계와 리스트가 표시됨

### 2) 콘텐츠 메이커 (Content Maker) - **핵심 화면**
- **목적**: AI를 활용한 광고주 맞춤형 블로그 포스팅 최종본 생성
- **사용자 입력값**: 
    - **기본**: 키워드, 업종/주제, 글 목적(정보/신뢰/홍보/방문), 말투, 이모지, 금지어
    - **광고주 특화**: 브랜드명/매장명, 지역, 핵심 강점(2~3개), 강조 포인트, CTA 방향
- **출력값**:
    - **[게시용 최종본]**: 제목 후보(3), 본문(실제 게시용), 해시태그(10)
    - **[SEO/AEO 점검]**: 단순 생성기가 아닌 블로그 운영 보조 시스템으로서, 검색 의도 기반 AEO 구조 확인. 첫 문단 핵심 답변, 질문형 소제목/결론 우선 문장, 구매/방문 전 확인 요소(누구에게 필요한지, 기준 등), FAQ 2~3개, 메인/보조 키워드 분리 적용 여부를 상세 표시. (상위노출 보장 등 표현 금지)
    - **[이미지 가이드]**: 본문 내 정확한 삽입 추천 위치(번호 표시) 및 컨셉이 포함된 이미지 추천 카드. 추천 카드는 본문 삽입 위치 번호와 명확히 연결하며, 목표 글자수에 따라 1200자 2개, 1500자 3개, 2000자 이상 4개를 추천.
    - **[전략 메모]**: 키워드 최적화 분석 및 작성 의도 요약
- **버튼 동작**:
    - [주제/제목 생성]: 입력값 기반 아이디어 도출
    - [본문 생성]: 선택된 제목으로 실제 게시용 글 작성
    - [복사 / 내 보관함 저장]: 복사 옵션을 3가지(본문만 복사 / 본문+이미지 표시 포함 복사 / 이미지 검색 키워드만 복사)로 제공하며, 게시용 최종본 클립보드 일괄 복사 및 임시 저장 지원
- **완료 조건**: 생성된 본문이 내부 메모 형태가 아닌 실제 포스팅 구조를 갖추며, 이미지 삽입 포인트가 명확히 제시됨. 초보자가 "이 문단 뒤에 이미지를 넣으면 되는구나"라고 바로 이해할 수 있도록 구성. 생성 완료 시 SEO/AEO 점검 패널에서 본문 흐름을 확인할 수 있고, 모바일에서도 결과 영역으로 시선이 자연스럽게 이동하도록 스크롤/UI 조정.

### 3) 내 보관함 (My Storage)
- **목적**: 작성된 초안들의 로컬 아카이빙 및 관리
- **사용자 입력값**: 검색어 (제목 기반)
- **출력값**: 저장된 초안 리스트 (제목, 날짜, 키워드 요약)
- **버튼 동작**:
    - [열기]: 해당 초안을 콘텐츠 메이커로 불러오기
    - [삭제]: 로컬 스토리지에서 해당 데이터 영구 삭제
- **완료 조건**: 로컬 스토리지에 저장된 JSON 데이터를 파싱하여 리스트를 정확히 렌더링함

### 4) 고급 설정 (Settings)
- **목적**: 시스템 환경 설정 및 확장성 확보
- **사용자 입력값**: Gemini API Key (관리자용), 블로그 기본 정보
- **출력값**: API 연결 상태, 현재 버전 정보
- **버튼 동작**:
    - [저장]: 설정값 로컬 저장
    - [연결 테스트]: API 키 유효성 검사
    - [댓글 자동화(슬롯)]: "준비 중" 메시지 표시 (1차 MVP 미구현)
- **완료 조건**: 설정 저장 시 시스템 전역에서 해당 API 키를 사용하여 통신 가능함

## 2. 콘텐츠 메이커 상세 명세 (MVP 핵심)

- **사용자 편의성 및 가독성(UI/UX) 개선 항목**:
    - 기능적 편의성:
        - 개요 소제목 수정/추가/삭제 지원
        - 주제 다시 생성 시 이전과 겹치지 않는 새로운 주제 후보 생성
        - 업체(브랜드)별 기본 입력값 저장 및 불러오기(프리셋) 지원
    - 시각적 가독성 (UI 뷰어 최적화):
        - 화면을 조잡하게 만드는 상단 단계 진행 박스(Stepper) 전면 제거 (작은 텍스트 인디케이터로 대체)
        - 입력 설명은 화면 직접 노출 대신 물음표(?) 툴팁으로 이동 (모바일에서는 짧고 직관적인 안내로 제공)
        - 선택/공란 가능/기본값 있음 등 부가 문구를 화면에서 최소화
        - 폼 구성을 '기본 입력'과 '고급 입력' 영역으로 분리
        - **모바일 우선 UX**: 입력값이 길게 먼저 나오는 것을 방지하고, 생성 완료 후 결과 영역으로 자연스럽게 이동하도록 동선 개선
        - 버튼 명칭 직관화: '로컬 저장' 등의 모호한 표현 대신 **'내 보관함 저장'** 등 사용자 친화적 문구 사용
        - **처음 사용 가이드 및 예시**: PC는 카드형, 모바일은 접힘형으로 제공. 키워드는 "내 글이 검색되길 원하는 핵심 단어"라는 설명 추가, 매장 홍보/상품 후기/정보글 등 상황별 예시 제공 (실제 업체/브랜드처럼 보이는 예시명 절대 사용 금지), '예시 자동 채우기' 버튼 제공
        - **본문 내 이미지 삽입 표시**: 게시용 본문 안에 `[이미지 삽입 추천 N]` 블록을 직접 삽입하고, 하단 이미지 추천 카드의 번호와 위치가 1:1로 연결되도록 표시
        - **SEO/AEO 점검 표시**: 검색 의도 기반 구조, 키워드 자연 반영, 답변형 문장 포함, 본문 흐름 점검, 이미지 삽입 위치 표시를 결과 영역에서 바로 확인
        - **조건부 FAQ**: 정보 전달, 상품 홍보, 비교형 제목처럼 자연스러운 글에만 FAQ 2~3개를 하단에 추가
        - 내부 메모는 접힘(아코디언) 영역 중심으로 정리하고, 사용자 검수용 점검은 별도 패널로 표시
- **입력 필드 상세**:
    - `Keyword`: 메인 키워드 (Input Text)
    - `Category`: 업종/주제 (Select Box)
    - `Goal`: 글 목적 (Radio: 정보 전달, 신뢰 형성, 상품 홍보, 방문 유도)
    - `Tone`: 말투 (Select: 친절한, 전문적인, 유머러스한 등)
    - `Brand/Region`: 브랜드명 및 지역 (Input Text)
    - `Strengths`: 핵심 강점 2~3개 (Textarea)
    - `Emphasis`: 꼭 강조할 포인트 (Input Text)
    - `CTA`: 마무리/행동 유도 방향 (Input Text)
- **목적별 템플릿 구조**:
    - **정보 전달형**: 문제 공감 → 핵심 정보 3가지 → 요약 및 도움말
    - **신뢰 형성형**: 업계 문제 제기 → 우리만의 원칙/과정 → 진심 어린 약속
    - **상품 홍보형**: 타겟 니즈 자극 → 상품 효과/특징 → 특별 혜택 및 유도
    - **방문 유도형**: 위치/첫인상 → 공간 경험/가치 → 지도 및 예약 안내
- **버튼 흐름**:
    - `단계별 생성` 로직(주제 → 제목 → 본문)은 유지하되, 거창한 단계 UI 대신 결과 영역에서 다음 액션을 자연스럽게 유도
    - 결과 확인 후 사용자가 에디터 클릭 시 → `상태: 수정 중`
    - `복사` 버튼 클릭 시 → 게시용 패키지(제목+본문+태그) 일괄 복사

## 3. 댓글 응답 관리 (Comment Management) 기획 방향
- **입력 방식 분리 고정**: 상단 탭이나 버튼으로 입력 방식(수동, 캡처)을 고정하여 혼란 방지
- **역할 명확화**: 수동 댓글 입력 / 캡처 이미지 업로드 / URL 자동화 브리지의 역할을 명확히 분리 (URL 자동화 브리지는 고급 기능/로컬 브리지 기능으로 분리)
- **수동 입력 단순화**: 작성자명은 필수가 아닌 선택값으로 변경, 댓글 내용만 넣어도 즉시 대댓글 생성이 가능하게 원스텝 연결
- **필수 입력 최소화**: 포스팅 제목의 필수 표시를 제거하고, 작성자명이 없을 때는 화면에서 `댓글 1`처럼 자동 표기
- **모바일 캡처 업로드**: 모바일 환경에서도 정상 작동하도록 캡처 이미지 업로드 및 수동 댓글 입력 호환성 확보
- **UI 단순화**: 핵심 기능 중심으로 버튼 간소화 및 '전체 상호대댓글 생성' 버튼의 명확한 작동
- **상호대댓글 품질 고도화 (템플릿 탈피)**:
    - 단순 템플릿 답변이 아닌, 댓글 내용(핵심 표현, 감정, 의도, 구매 의사, 경험 공유 등)에 직접 반응하는 대댓글 생성
    - 하얀가루, 가루날림, 백탁, 떡짐, 귀차니즘, 편리함, 구매 의사처럼 원댓글에 나온 핵심 표현을 우선 포착
    - 물음표만으로 질문형으로 오판하지 않고 실제 질문 의도가 있을 때만 질문형으로 처리
    - 질문형이 아닌 댓글에 “문의하신 부분은…”, “방문 목적에 맞춰…”, “먼저 확인해보시면 좋습니다” 같은 기계적 fallback 문구가 나오지 않도록 엄격히 통제
    - 댓글 유형을 세분화 (제품고민공감형, 사용감반응형, 구매의사형, 기대감표현형, 경험공유형 등)하여 맞춤 대응
- **Variation 확보**: 다시 생성 시, 이전 결과와 확실한 차이가 있는 새로운 문장의 답변 도출

## 4. 판매용 접속 제어 (Access Gate)

- **사용권 표시**: 앱 메뉴 하단에 사용권 라벨, 만료일, 코드 초기화 버튼 표시
- **보안 안내**: 프론트 기반 간단 인증이므로 완벽한 보안 목적이 아니며, 정식 판매 확대 시 서버 검증 권장

---

## 5. 상태 흐름 정의 (State Transitions)

1. **[입력 전]**: 입력 필드가 비어있고 '생성' 버튼 비활성화
2. **[입력 완료]**: 필수 입력값 충족 시 '생성' 버튼 활성화
3. **[생성 중]**: 로딩 애니메이션 표시, 버튼 비활성화
4. **[생성 완료]**: 결과창에 텍스트 렌더링, '복사/저장' 버튼 노출
5. **[수정 중]**: 결과창 에디터 포커스 시 상태 변경, 변경 사항 자동 저장 준비
6. **[저장됨]**: 사용자가 명시적으로 저장하거나 자동 저장 완료 시 로컬 데이터 업데이트
7. **[복사 완료]**: 클립보드 복사 성공 시 성공 토스트 메시지 출력

## 6. 코덱스(Codex) 인계용 체크리스트

### ❶ 기술 구조 및 환경
- [ ] 폴더 구조: `/frontend`, `/backend`, `/shared` 기반 구성
- [ ] 메뉴 구조: 대시보드 / 콘텐츠 메이커 / 내 보관함 / 고급 설정

## 7. 1차 베타 피드백 기반 주요 개선 사항 (구현 완료)

1. **콘텐츠 메이커 가이드**: 키워드 입력 이해를 돕기 위한 "처음 사용 가이드" 및 "예시 자동 채우기" 제공
2. **이미지 위치 표시**: 게시용 본문 안에 `[이미지 삽입 추천 N]` 표시를 직접 삽입해 이미지 삽입 위치 직관적 제공
3. **추천 카드 연동**: 하단 이미지 추천 카드와 본문 표시 번호를 1:1로 매칭
4. **복사 옵션 분리**: '본문만', '본문+이미지 표시', '이미지 키워드만' 3가지 복사 옵션 지원
5. **SEO/AEO 점검 강화**: 검색 의도 기반 구조, 키워드 자연 반영, 답변형 문장 포함, 본문 흐름 점검, 이미지 삽입 위치 표시 여부를 결과 영역에서 검수
6. **AEO형 답변 구조**: 첫 문단 핵심 답변, 소제목별 결론형 첫 문장, 정보/상품/비교형 글의 조건부 FAQ 2~3개 생성
7. **대댓글 품질 고도화**: 원댓글의 핵심 단어와 감정/의도를 파악해 다채로운 상호대댓글 생성 (단순 템플릿 답변 탈피)
8. **수동 입력 편의성**: 댓글 작성자명을 선택값으로 변경, 미입력 시에도 댓글 내용만으로 정상 동작 지원
9. **재생성 다변화(Variation)**: 대댓글 '다시 생성' 시 기존 생성된 문장과 다른 variation이 확실히 도출되도록 반영
10. **접속코드 게이트**: 정적 베타/1차 판매용 접속코드와 만료일 기반 간단 인증 추가
11. **정적 베타 유지**: URL 기반 자동 수집/등록 없이도 콘텐츠 메이커 및 수동/캡처 기반 댓글 관리 기능 전체 동작 보장
12. **확장성 모듈화**: URL 자동 수집 및 등록은 로컬 `blog-automation` 브리지 기반의 고급 기능으로 분리/별도 관리
