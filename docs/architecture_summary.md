# [확정안] 블로그 올인원 운영 시스템 구조 정의서

본 문서는 '블로그 올인원 운영 시스템'의 1차 MVP 개발을 위한 최종 구조와 방향을 정의합니다.

## 1. 프로젝트 핵심 원칙
- **상품성**: 크몽 판매를 고려한 '관리형/반자동형' 솔루션 지향
- **효율성**: 화려함보다 직관적인 동선과 빠른 로딩 속도 우선
- **확장성**: 기존 댓글 자동화 기능을 모듈로 연결할 수 있는 브릿지 구조

---

## 2. MVP 1차 기능 범위
- **콘텐츠 메이커**: 키워드 기반 5대 주제별 포스팅 생성 (내부적으로 SEO/AEO/GEO 최적화 구조를 유지하되, 화면 용어는 "생성 결과", "블로그 원고 결과", "글 발행 전 확인사항"과 같이 직관적이고 쉬운 용어를 사용하여 사용자 노출을 축소, 제목/개요/본문/해시태그 생성). 친근한 말투(했어요, 더더라고요 등 구어체 중심)를 공통으로 적용하는 고도화된 말투 엔진을 사용합니다.
- **글 방향(주제) 및 글 순서 정하기 연동**: 생성된 추천 주제(글 방향) 후보를 사용자가 직접 수정할 수 있으며, 이 수정본이 제목 생성, 글 순서 정하기, 본문 생성에 실시간으로 반영됩니다. 또한 제목 생성 후 도출된 개요 소제목을 직접 수정, 추가, 삭제할 수 있으며, Drag & Drop 및 모바일용 위/아래 이동 버튼을 통해 순서를 변경("글 순서 정하기")합니다. 소제목별 메모 입력란에 **"이 부분에 꼭 넣고 싶은 내용을 적어주세요. 안 적어도 괜찮아요. (선택)"**과 같은 쉬운 문구를 사용하여 선택 입력임을 명확히 노출하고, 수정한 개요와 메모가 본문 생성 시 프롬프트에 자동 주입되어 본문에 반영되는 아키텍처를 가집니다.
- **주제별 결과 차별화 엔진**: 정보형, 후기형, 비교형, 방문형, 상품형의 5대 주제별로 특화된 프롬프트 템플릿과 구조 제어 로직을 적용하여, 동일 키워드라도 도입부(첫 문단), 소제목, 결론부 구조가 확실히 차별화되도록 빌드합니다.
- **상품 후기 메이커**: 상품 상세 이미지 여러 장(업로드/Ctrl+V 누적 지원) 및 정보 다중 추출, 사용자 메모 기반으로 후기형 블로그 글 초안 자동 생성 (도입부~구매 전 확인점 구조화, 내부 라벨 노출 차단, 건강/효능 과장 표현 금지)
- **AI 주제/제목 추천**: 사용자 키워드 기반 트렌디한 주제 및 클릭을 부르는 제목 후보 제안. 기본 제목 후보는 검색형, 후기형, 클릭형 중심의 3개 후보를 우선 노출하고 비교형, 선택형 등 추가 후보는 **"제목 더 보기"** 버튼으로 펼쳐 확인 가능하게 UI를 구현합니다.
- **블로그 원고 결과 (14종 원고 패키지)**: 메인/보조 키워드, 검색 의도 분석, 홈피드 클릭 포인트, 제목 후보 5개, 최종 추천 제목, 첫 문장 후보, 블로그 본문, 사진 배치 가이드, 업체/상품 정보 정리, 이런 분께 추천해요, FAQ, 해시태그, 최종 검수표로 이루어진 구조화된 JSON 응답 스키마와 결과 UI 컴포넌트. 화면에서는 **선택 제목 바로 아래에 본문 원고를 즉시 노출**하고, 해시태그, 사진 배치 가이드, 추천 대상, FAQ, 글 발행 전 확인사항 순으로 배치합니다. 메인/보조 키워드 및 검색 의도 분석 데이터는 하단 접힘 영역(**'자세한 글 분석 보기'** 또는 **'글 작성 참고 정보'**)으로 이동하여 렌더링됩니다.
- **카테고리별 필수 확인 기준 검증**: 맛집, 카페, 아이와 갈만한 곳, 상품 리뷰, 생활용품/계절템, 체험단/협찬 리뷰별 필수 항목(주차, 가격, 영업시간, 웨이팅, 사용 기간, 제품 효과 등)의 기입 여부를 생성 엔진 및 프론트 검수에서 실시간으로 대조합니다. 정보가 누락된 경우 AI 임의 조작을 차단하고 `[확인 필요]` fallback 문구를 삽입합니다. 협찬 여부 미기입 시 최종 검수표(글 발행 전 확인사항)에 `[협찬 여부 확인 필요]` 경고를 강제 렌더링합니다.
- **글 발행 전 확인사항 (블로그 글 구조 점검)**: '상위노출 보장' 등의 표현을 배제한 **'블로그 운영 보조 시스템'**입니다. 제목(30자 내외, 메인 키워드 포함, 클릭 이유, 금칙어 제한), 첫 문단(첫 문장 키워드, 2~3회 반복, 정보형 시작, 광고 느낌 금지), 본문(결론형 답변 시작, 경험 기반 전개, FAQ/질문형 구조, 업체/상품 정보 하단 정리) 기준에 맞춰 검색 의도 기반 글 구성을 돕습니다. 점검 패널에서는 제목/첫 문장 키워드 반영, 첫 문단 키워드 반복, 보조 키워드 분산, 제목/본문 일치, 의도/목적 일치, 금칙어 및 과장 표현 없음 등을 확인합니다.
- **이미지 가이드 시스템**: 본문 흐름에 최적화된 이미지 삽입 위치 및 컨셉 추천. 하단 추천 카드와 본문 내 삽입 위치를 명확히 연결하며, 이미지 컨셉, 직접 촬영 추천(우선), AI 이미지 프롬프트, 이미지 사이트 검색어 및 복사 버튼을 제공합니다. 목표 글자수에 따라 추천 수를 조정하고 복사 옵션을 3가지로 세분화.
- **작성 프로필 (브랜드/상품 프로필) 관리**: 품목/브랜드별로 자주 쓰는 사용자 유형, 업종/주제, 글 목적, 말투, 금지어, CTA 톤, 목표 글자수, 브랜드명/매장명, 지역, 핵심 강점, 강조 포인트, 주소, 영업시간, 가격/가격대, 구매처, 문의 방법, 배송/교환 정보를 여러 개 저장하고 선택하여 불러옵니다.
- **판매용 접속 제어 (1차)**: 크몽/직접판매 환경에서 URL 무단 공유를 막기 위해 접속코드(예: `MGO-TEST7`/7일 테스트, `MGO-BETA30`/30일 사용권), 만료일, active 여부 확인. `localStorage` 인증을 유지하되 만료 시 접속을 제한합니다. (단, 정적 베타 프론트 인증이므로 정식 확대 시 Supabase/Cloudflare Workers 기반 서버 검증 권장)
- **스마트 태그**: 10~15개의 해시태그를 메인 키워드, 보조 키워드, 지역명+카테고리, 카테고리+후기, 브랜드/상품명, 검색 의도형 롱테일 기준으로 생성하며, 네이버 블로그에 바로 붙여넣기 편하도록 전체 복사 및 그룹별 복사 기능 제공
- **댓글 응답 관리**: 수동 입력 또는 캡처 업로드 기반으로 원댓글의 핵심 표현, 감정, 의도, 구매 의사를 반영한 상호대댓글 초안 생성. 작성자명은 선택값이며 URL 자동 수집/자동 등록은 로컬 브리지 기반 고급 기능으로 분리.
- **작업 대시보드**: 최근 작업 리스트 확인 및 연속성 있는 작업 환경 제공
- **사용자 편의성 및 가독성(UI/UX) 강화**: 
    - 기능 고도화: 글 방향(주제) 직접 수정 기능, 글 순서 정하기 및 메모 편집 컴포넌트, 카테고리 누락 검증 로직, 결과 가공 필터.
    - 인터페이스 최적화: 
        - 툴팁 기반의 입력 설명 제공 (모바일에서는 짧고 직관적인 안내 적용) 및 부가 안내 문구 최소화
        - 기본 입력과 고급 입력의 시각적 분리
        - **모바일 우선 UX**: 입력값이 길게 먼저 나오는 것을 방지. 상품 후기 메이커의 경우, 긴 이미지 추출/보정 영역은 기본 접힘 처리(이미지 없을 시 안내 문구만 표시)하고 이미지 등록 시 자동 펼침. OCR 원문은 보조 접힘 영역으로 분리하여 생성 버튼까지의 동선을 최단거리로 유지하며 다중 이미지 미리보기를 모바일 환경에 맞게 최적화함. 생성 완료 후 결과 영역으로 자연스럽게 이동.
        - 분석 정보는 하단 접힘(**'자세한 글 분석 보기'** 또는 **'글 작성 참고 정보'**) 뷰로 분리하고, 사용자 검수용 글 발행 전 확인사항(블로그 글 구조 점검)은 결과 영역 하단에 배치
        - **처음 사용 가이드/키워드 가이드**: PC는 카드형, 모바일은 접힘형으로 제공. "내 글이 검색되길 원하는 핵심 단어"라는 설명과 함께 중립적인 범용 예시(예: 수분크림, 무선 미니청소기, 아기 식판, 단백질 쉐이크, 여름 샌들 등)를 제공하며, 특정 브랜드명 노출을 배제하고 '예시 자동 채우기' 버튼을 적용

## 3. 메뉴별 핵심 기능 (1줄 설명)
- **대시보드**: 운영 현황과 최근 작성된 콘텐츠를 한눈에 관리하는 관제탑
- **콘텐츠 메이커**: 아이디어 도출부터 초안 완성까지의 전 과정을 수행하는 핵심 작업실 (공통 말투 엔진 사용)
- **상품 후기 메이커**: 상세 이미지 다중 업로드/정보 추출과 사용자 메모만으로 후기형 포스팅을 자동 완성하는 특화 작업실
- **내 보관함**: 생성된 콘텐츠 자산을 안전하게 보관하고 필요할 때 꺼내 쓰는 창고
- **고급 설정**: API 연동 관리 및 추후 댓글 자동화 기능을 연결하는 확장 허브

## 4. MVP 화면 구성안 (UI Layout)
- **공통**: 사이드바 형태의 내비게이션 (대시보드 / 메이커 / 보관함 / 설정)
- **대시보드**: 
    - [상단] 누적 생성 콘텐츠 수, 오늘 남은 AI 사용량 등 요약 위젯
    - [하단] 최근 편집 중인 초안 리스트 (최근 5개)
- **콘텐츠 메이커**: 
    - [좌측 40%] 입력 영역: 기본 설정 + 광고주 특화 정보(브랜드, 강점, CTA) 입력 폼. 추가적으로 글 방향(주제) 선택/수정 기능 및 '글 구성 만들기'로 도출된 소제목들을 제어하는 **'글 순서 정하기' 컴포넌트**(Drag & Drop 및 모바일용 위/아래 이동 버튼 적용, 소제목별 '이 부분에 꼭 넣고 싶은 내용을 적어주세요. 안 적어도 괜찮아요. (선택)' 개별 메모 입력 필드 포함)가 위치함.
    - [우측 60%] 결과 영역: 
        - 상단: 심플한 텍스트형 상태 인디케이터
        - 중단: **생성 결과 (블로그 원고 결과)** (선택 제목 바로 아래 본문 원고를 즉시 렌더링. 그 아래 해시태그, 사진 배치 가이드, 추천 대상, FAQ, 글 발행 전 확인사항(검수표)을 차례로 배치). 본문 내 소제목은 CSS로 굵게 표시되나, 우측의 `복사` 버튼 클릭 시 `*`, `**`, `###` 등 마크다운 기호가 완전히 필터링된 순수 텍스트(줄바꿈 유지) 상태로 복사됨.
        - 검수: **글 발행 전 확인사항 (블로그 글 구조 점검 패널)** 및 **카테고리별 필수 항목 검수표** (누락 정보가 발생한 경우 `[확인 필요]`로 명시되며, 협찬 여부가 누락된 경우 최종 검수표에 `[협찬 여부 확인 필요]` 경고가 렌더링됨)
        - 하단: 본문 내 번호와 연결된 이미지 추천 카드. 실무 중심으로 이미지 컨셉, 직접 촬영 추천, AI 프롬프트, 검색어 및 복사 버튼 제공. 3가지 복사 옵션 버튼 제공.
        - 접힘: 메인 키워드, 보조 키워드, 검색 의도 분석 및 홈피드 클릭 포인트 등 분석 정보는 하단 접힘 영역(**'자세한 글 분석 보기'** 또는 **'글 작성 참고 정보'**)으로 격리하여 초보자의 화면 혼란 방지.
- **접속코드 화면**:
    - 앱 첫 진입 시 접속코드 입력 화면 표시
    - 유효 코드 인증 후 콘텐츠 메이커, 댓글 응답 관리, 내 보관함, 고급 설정 접근 허용
    - 좌측 메뉴 하단에 사용권 라벨과 만료일 표시, 코드 초기화 버튼 제공
- **댓글 응답 관리**:
    - [수동 입력] 작성자명은 선택값이며, 미입력 시 댓글 1처럼 자동 표기
    - [캡처 업로드] OCR 원문과 댓글 카드 분리, 수동 보정 후 대댓글 생성
    - [생성 결과] 댓글 유형(제품고민공감형, 사용감반응형, 구매의사형, 기대감표현형, 경험공유형 등), 핵심 표현, 감정/의도를 반영한 대댓글과 복사 버튼 제공
- **내 보관함**: 
    - 카드 또는 리스트 뷰: 제목, 작성일, 요약 내용 표기
- **고급 설정**: 
    - 서비스 설정 카드: Gemini API Key, 블로그 닉네임 설정
    - 확장 기능 카드: URL 자동 댓글 수집/자동 등록 등은 로컬 `blog-automation` 브리지 연결 기반의 **고급 기능**으로 완전히 분리하여 제공

## 5. 사용자 흐름 (User Flow)
1. **키워드 투입**: 사용자가 작성하고 싶은 핵심 키워드 입력
2. **글 방향(주제) 선택 및 직접 수정**: AI가 추천한 주제(글 방향) 후보를 확인하고 사용자가 직접 수정하거나 그대로 선택
3. **제목 및 글 구성 만들기**: 수정된 글 방향을 기반으로 제목 후보를 생성(검색/후기/클릭형 3종 우선 노출, 추가 후보는 '제목 더 보기' 펼침)하고 소제목을 생성하여 '글 순서 정하기'로 순서를 조정 및 개별 메모("이 부분에 꼭 넣고 싶은 내용을 적어주세요. 안 적어도 괜찮아요. (선택)")를 입력
4. **본문 및 결과 생성**: '이 구성으로 본문 만들기'를 클릭하여 본문 원고를 실시간 생성. 결과 화면에서 선택 제목 바로 아래 본문 원고가 즉시 노출되고 해시태그, 이미지 가이드, FAQ, 발행 전 확인사항이 하단 순서대로 나열됨
5. **이미지/복사 선택**: 본문 내 `[여기에 이미지를 넣어주세요 이미지 N]` 표시와 하단 이미지 추천 카드를 맞춰 보고, 본문만/본문+이미지 표시/이미지 검색어만 중 필요한 복사 옵션 선택
6. **글 구성 체크**: '글 발행 전 확인사항' 패널에서 제목, 첫 문단, 소제목, 키워드, 마무리 문장 등이 검색에 잘 맞는지 최종 점검
7. **발행 완료**: 제목, 본문, 태그를 각각 '원클릭 복사'하여 블로그에 업로드

## 6. 1차 개발 순서 (Roadmap)
1. **[Phase 1] 시스템 환경 구축**: React(Vite) + Node.js 기초 프레임워크 및 라우팅 설정
2. **[Phase 2] 콘텐츠 메이커 UI**: 더미 데이터를 활용한 '입력-결과' 2분할 인터페이스 구현
3. **[Phase 3] AI 엔진 연동**: Gemini API를 이용한 주제/본문/태그 생성 로직 개발 및 연결
4. **[Phase 4] 로컬 데이터 관리**: 생성된 초안을 저장하고 불러오는 보관함 및 대시보드 연동
5. **[Phase 5] 최종 폴리싱 및 추가 UX 피드백 반영 완료**:
    - 콘텐츠 메이커 온보딩(처음 사용 가이드, 예시 자동 채우기) 구현
    - 작성 프로필 (브랜드/상품 프로필) 다중 저장/불러오기 적용
    - 본문 내 `[여기에 이미지를 넣어주세요 이미지 N]` 표시 및 번호 연결, 3분할 복사 옵션 적용
    - 글 발행 전 확인사항 (블로그 글 구조 점검) 패널 및 조건부 FAQ 2~3개 생성 로직 적용
    - 이미지 추천 카드 실무형 개편 (컨셉, 직접 촬영 추천, AI 이미지 프롬프트, 검색어, 복사 버튼)
    - 전체/그룹별 해시태그 복사(네이버 블로그 붙여넣기 최적화) 기능 추가
    - 템플릿 탈피형 맞춤 상호대댓글 생성 고도화 및 질문형 오판 방지(금지어 차단)
    - 수동 댓글 작성자명 선택화 등 편의성 강화 및 variation 알고리즘 보강
    - 정적 베타 모드 단독 구동 보장, 접속코드+만료일 게이트 적용, URL 자동화 기능의 로컬 브리지 분리 확정
    - **글 방향 직접 수정 기능 및 연동**: 글 방향을 직접 편집하여 제목/소제목/본문 생성에 프롬프트 반영
    - **쉬운 화면 용어 및 순서 재배치**: 제목 바로 아래 본문 원고를 배치하고 해시태그, 가이드, FAQ, 발행 전 확인사항 순서로 노출하며 키워드/검색 의도 분석 정보는 하단 접힘('자세한 글 분석 보기' / '글 작성 참고 정보')으로 재배치
    - **제목 후보 UI 최적화**: 기본 제목 후보 3종 우선 노출 및 '제목 더 보기' 펼침 버튼 적용
    - **메모 문구 완화 및 라벨 개편**: 메모 필드 텍스트 변경 및 버튼/라벨 쉬운 말("글 순서 정하기", "글 구성 만들기", "항목 추가", "이 구성으로 본문 만들기") 적용
