# [확정안] 블로그 올인원 운영 시스템 구조 정의서

본 문서는 '블로그 올인원 운영 시스템'의 1차 MVP 개발을 위한 최종 구조와 방향을 정의합니다.

## 1. 프로젝트 핵심 원칙
- **상품성**: 크몽 판매를 고려한 '관리형/반자동형' 솔루션 지향
- **효율성**: 화려함보다 직관적인 동선과 빠른 로딩 속도 우선
- **확장성**: 기존 댓글 자동화 기능을 모듈로 연결할 수 있는 브릿지 구조

---

## 2. MVP 1차 기능 범위
- **AI 주제/제목 추천**: 사용자 키워드 기반 트렌디한 주제 및 클릭을 부르는 제목 후보 제안
- **광고주 맞춤 본문 메이커**: 목적별(정보/신뢰/홍보/방문) 최적화 템플릿 기반의 게시용 본문 자동 생성
- **이미지 가이드 시스템**: 본문 흐름에 최적화된 이미지 삽입 위치 및 컨셉 추천
- **스마트 태그**: 포스팅 문맥 및 브랜드 키워드에 최적화된 네이버 해시태그 패키지 구성
- **작업 대시보드**: 최근 작업 리스트 확인 및 연속성 있는 작업 환경 제공

---

## 3. 메뉴별 핵심 기능 (1줄 설명)
- **대시보드**: 운영 현황과 최근 작성된 콘텐츠를 한눈에 관리하는 관제탑
- **콘텐츠 메이커**: 아이디어 도출부터 초안 완성까지의 전 과정을 수행하는 핵심 작업실
- **내 보관함**: 생성된 콘텐츠 자산을 안전하게 보관하고 필요할 때 꺼내 쓰는 창고
- **고급 설정**: API 연동 관리 및 추후 댓글 자동화 기능을 연결하는 확장 허브

---

## 4. MVP 화면 구성안 (UI Layout)
- **공통**: 사이드바 형태의 내비게이션 (대시보드 / 메이커 / 보관함 / 설정)
- **대시보드**: 
    - [상단] 누적 생성 콘텐츠 수, 오늘 남은 AI 사용량 등 요약 위젯
    - [하단] 최근 편집 중인 초안 리스트 (최근 5개)
- **콘텐츠 메이커**: 
    - [좌측 40%] 입력 영역: 기본 설정 + 광고주 특화 정보(브랜드, 강점, CTA) 입력 폼
    - [우측 60%] 결과 영역: 
        - 상단: 단계별 선택기 (주제/제목)
        - 중단: **게시용 최종본** (에디터 형태, 복사 강조)
        - 하단: 이미지 추천 카드 및 전략 메모 (아코디언/탭 구조)
- **내 보관함**: 
    - 카드 또는 리스트 뷰: 제목, 작성일, 요약 내용 표기
- **고급 설정**: 
    - 서비스 설정 카드: Gemini API Key, 블로그 닉네임 설정
    - 확장 기능 카드: [고급] 댓글/대댓글 자동화 모듈 연결 버튼 (비활성화 상태)

---

## 5. 사용자 흐름 (User Flow)
1. **키워드 투입**: 사용자가 작성하고 싶은 핵심 키워드 입력
2. **주제 선택**: AI가 제안한 5가지 주제 중 마음에 드는 것을 사용자가 '직접 클릭'
3. **초안 생성**: 선택된 주제를 바탕으로 AI가 제목과 본문 초안을 실시간 생성
4. **검토 및 수정**: 생성된 내용을 에디터에서 사용자가 확인하고 필요 시 문구 수정
5. **발행 완료**: 제목, 본문, 태그를 각각 '원클릭 복사'하여 블로그에 업로드

---

## 6. 1차 개발 순서 (Roadmap)
1. **[Phase 1] 시스템 환경 구축**: React(Vite) + Node.js 기초 프레임워크 및 라우팅 설정
2. **[Phase 2] 콘텐츠 메이커 UI**: 더미 데이터를 활용한 '입력-결과' 2분할 인터페이스 구현
3. **[Phase 3] AI 엔진 연동**: Gemini API를 이용한 주제/본문/태그 생성 로직 개발 및 연결
4. **[Phase 4] 로컬 데이터 관리**: 생성된 초안을 저장하고 불러오는 보관함 및 대시보드 연동
5. **[Phase 5] 최종 폴리싱**: 직관적인 사용자 경험(UX) 피드백 반영 및 버그 수정
