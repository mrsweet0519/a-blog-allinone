# [확정안] 블로그 올인원 운영 시스템 구조 정의서

본 문서는 '블로그 올인원 운영 시스템'의 1차 MVP 개발을 위한 최종 구조와 방향을 정의합니다.

## 1. 프로젝트 핵심 원칙
- **상품성**: 크몽 판매를 고려한 '관리형/반자동형' 솔루션 지향
- **효율성**: 화려함보다 직관적인 동선과 빠른 로딩 속도 우선
- **확장성**: 기존 댓글 자동화 기능을 모듈로 연결할 수 있는 브릿지 구조

---

## 2. MVP 1차 기능 범위
- **AI 주제/제목 추천**: 사용자 키워드 기반 트렌디한 주제 및 클릭을 부르는 제목 후보 제안
- **광고주 맞춤 본문 메이커**: 목적별(정보/신뢰/홍보/방문) 최적화 템플릿 기반의 게시용 본문 자동 생성
- **이미지 가이드 시스템**: 본문 흐름에 최적화된 이미지 삽입 위치(번호 표시) 및 컨셉 추천. 추천 카드와 본문 내 삽입 위치를 명확히 연결하며, 목표 글자수에 따라 추천 수를 조정(1200자 2개, 1500자 3개, 2000자 이상 4개)하고 복사 옵션을 3가지(본문만, 본문+이미지 삽입 표시 포함, 이미지 검색 키워드만)로 세분화.
- **스마트 태그**: 포스팅 문맥 및 브랜드 키워드에 최적화된 네이버 해시태그 패키지 구성
- **댓글 응답 관리**: 수동 입력 또는 캡처 업로드 기반으로 원댓글의 핵심 표현, 감정, 의도, 구매 의사를 반영한 상호대댓글 초안 생성. 작성자명은 선택값이며 URL 자동 수집/자동 등록은 로컬 브리지 기반 고급 기능으로 분리.
- **작업 대시보드**: 최근 작업 리스트 확인 및 연속성 있는 작업 환경 제공
- **사용자 편의성 및 가독성(UI/UX) 강화**: 
    - 기능 고도화: 개요 소제목 관리, 주제 재생성 최적화, 업체별 프리셋 저장 기능
    - 인터페이스 최적화: 
        - 툴팁 기반의 입력 설명 제공 (모바일에서는 짧고 직관적인 안내 적용) 및 부가 안내 문구 최소화
        - 기본 입력과 고급 입력의 시각적 분리
        - **모바일 우선 UX**: 입력값이 길게 먼저 나오는 것을 방지하고 생성 완료 후 결과 영역으로 자연스럽게 이동하도록 흐름 최적화
        - 내부 메모/최적화 체크 영역의 접힘(아코디언) 뷰 적용
        - **처음 사용 가이드/키워드 가이드**: PC는 카드형, 모바일은 접힘형으로 제공. "내 글이 검색되길 원하는 핵심 단어"라는 설명과 함께 상황별(매장 홍보, 상품 후기, 정보글) 중립적 예시 제공 (실제 업체명 예시 배제) 및 '예시 자동 채우기' 버튼 적용
---

## 3. 메뉴별 핵심 기능 (1줄 설명)
- **대시보드**: 운영 현황과 최근 작성된 콘텐츠를 한눈에 관리하는 관제탑
- **콘텐츠 메이커**: 아이디어 도출부터 초안 완성까지의 전 과정을 수행하는 핵심 작업실
- **내 보관함**: 생성된 콘텐츠 자산을 안전하게 보관하고 필요할 때 꺼내 쓰는 창고
- **고급 설정**: API 연동 관리 및 추후 댓글 자동화 기능을 연결하는 확장 허브

---

## 4. MVP 화면 구성안 (UI Layout)
- **공통**: 사이드바 형태의 내비게이션 (대시보드 / 메이커 / 보관함 / 설정)
- **대시보드**: 
    - [상단] 누적 생성 콘텐츠 수, 오늘 남은 AI 사용량 등 요약 위젯
    - [하단] 최근 편집 중인 초안 리스트 (최근 5개)
- **콘텐츠 메이커**: 
    - [좌측 40%] 입력 영역: 기본 설정 + 광고주 특화 정보(브랜드, 강점, CTA) 입력 폼
    - [우측 60%] 결과 영역: 
        - 상단: 심플한 텍스트형 상태 인디케이터 (시각적 피로도를 높이는 단계별 박스 UI 제거)
        - 중단: **게시용 최종본** (에디터 형태, 본문 안에 `[이미지 삽입 추천 N]` 블록을 직접 삽입해 어느 문장 뒤에 이미지를 넣을지 표시)
        - 하단: 본문 내 번호와 연결된 이미지 추천 카드 및 전략 메모 (아코디언/탭 구조), 3가지 복사 옵션(본문만, 본문+이미지 표시 포함, 이미지 검색 키워드만) 버튼 및 '내 보관함 저장' 버튼
- **댓글 응답 관리**:
    - [수동 입력] 작성자명은 선택값이며, 미입력 시 댓글 1처럼 자동 표기
    - [캡처 업로드] OCR 원문과 댓글 카드 분리, 수동 보정 후 대댓글 생성
    - [생성 결과] 댓글 유형(제품고민공감형, 사용감반응형, 구매의사형, 기대감표현형, 경험공유형 등), 핵심 표현, 감정/의도를 반영한 대댓글과 복사 버튼 제공
- **내 보관함**: 
    - 카드 또는 리스트 뷰: 제목, 작성일, 요약 내용 표기
- **고급 설정**: 
    - 서비스 설정 카드: Gemini API Key, 블로그 닉네임 설정
    - 확장 기능 카드: URL 자동 댓글 수집/자동 등록 등은 로컬 `blog-automation` 브리지 연결 기반의 **고급 기능**으로 완전히 분리하여 제공

## 5. 사용자 흐름 (User Flow)
1. **키워드 투입**: 사용자가 작성하고 싶은 핵심 키워드 입력
2. **주제 선택**: AI가 제안한 5가지 주제 중 마음에 드는 것을 사용자가 '직접 클릭'
3. **초안 생성**: 선택된 주제를 바탕으로 AI가 제목과 본문 초안을 실시간 생성
4. **검토 및 수정**: 생성된 내용을 에디터에서 사용자가 확인하고 필요 시 문구 수정
5. **이미지/복사 선택**: 본문 내 `[이미지 삽입 추천 N]` 표시와 하단 이미지 추천 카드를 맞춰 보고, 본문만/본문+이미지 표시/이미지 키워드만 중 필요한 복사 옵션 선택
6. **발행 완료**: 제목, 본문, 태그를 각각 '원클릭 복사'하여 블로그에 업로드

---

## 6. 1차 개발 순서 (Roadmap)
1. **[Phase 1] 시스템 환경 구축**: React(Vite) + Node.js 기초 프레임워크 및 라우팅 설정
2. **[Phase 2] 콘텐츠 메이커 UI**: 더미 데이터를 활용한 '입력-결과' 2분할 인터페이스 구현
3. **[Phase 3] AI 엔진 연동**: Gemini API를 이용한 주제/본문/태그 생성 로직 개발 및 연결
4. **[Phase 4] 로컬 데이터 관리**: 생성된 초안을 저장하고 불러오는 보관함 및 대시보드 연동
5. **[Phase 5] 최종 폴리싱 (1차 베타 피드백 반영 완료)**:
    - 콘텐츠 메이커 온보딩(처음 사용 가이드, 예시 자동 채우기) 구현
    - 본문 내 `[이미지 삽입 추천 N]` 표시 및 번호 연결, 3분할 복사 옵션 적용
    - 템플릿 탈피형 맞춤 상호대댓글 생성 고도화 및 질문형 오판 방지(금지어 차단)
    - 수동 댓글 작성자명 선택화 등 편의성 강화 및 variation 알고리즘 보강
    - 정적 베타 모드 단독 구동 보장 및 URL 자동화 기능의 로컬 브리지 분리 확정
