# [계획서] 블로그 올인원 운영 시스템 (Blog All-in-One System)

본 프로젝트는 블로거와 소상공인을 위한 통합 운영 솔루션으로, AI 기반의 콘텐츠 생산과 관리 자동화를 목표로 합니다. 특히 크몽 판매용 상품으로서 '프리미엄한 디자인'과 '사용자 편의성'을 최우선으로 합니다.

## User Review Required

> [!IMPORTANT]
> **MVP 범위 확인**: 초기 단계에서 모든 자동화 기능을 넣기보다, 핵심 가치인 '글쓰기 생산성'에 집중할 예정입니다. 아래 제안된 MVP 범위가 판매 전략과 일치하는지 확인 부탁드립니다.

> [!NOTE]
> **디자인 테마**: '안티그래비티'의 정체성을 담아 다크 모드 기반의 세련된 글래스모피즘(Glassmorphism) 스타일을 제안합니다.

## 1. MVP 기능 범위 (1차 고정)

사용자가 '직접 관리하는 재미'를 느끼면서도 '반자동'으로 생산성을 높일 수 있는 핵심 기능에 집중합니다.

- **콘텐츠 메이커**: 키워드 기반 5대 주제별 포스팅 생성 (내부적으로 SEO/AEO/GEO 최적화 구조를 유지하되, 화면 용어는 "생성 결과", "블로그 원고 결과", "글 발행 전 확인사항"과 같이 직관적이고 쉬운 용어를 사용하여 사용자 노출을 축소, 제목/개요/본문/해시태그 생성). 친근한 말투(했어요, 더더라고요 등 구어체 중심)를 공통으로 적용하는 고도화된 말투 엔진을 사용합니다.
- **글 방향(주제) 및 글 순서 정하기 연동**: 생성된 추천 주제(글 방향) 후보를 사용자가 직접 수정할 수 있으며, 이 수정된 텍스트가 제목 생성, 글 순서 정하기, 본문 생성에 실시간 반영됩니다. 또한 제목 선택 후 도출된 개요 소제목을 직접 수정, 추가("항목 추가"), 삭제할 수 있으며, Drag & Drop 및 모바일용 위/아래 이동 버튼을 통해 순서를 변경("글 순서 정하기")합니다. 소제목별 메모 입력란에 **"이 부분에 꼭 넣고 싶은 내용을 적어주세요. 안 적어도 괜찮아요. (선택)"**과 같은 쉬운 문구를 사용하여 선택 입력임을 명확히 노출하고, 수정한 개요와 메모가 본문 생성 프롬프트에 자동 반영됩니다.
- **주제별 결과 차별화 엔진**: 정보형, 후기형, 비교형, 방문형, 상품형의 5대 주제별로 특화된 프롬프트 템플릿과 구조 제어 로직을 적용하여, 동일 키워드라도 도입부(첫 문단), 소제목, 결론부 구조가 확실히 차별화되도록 빌드합니다.
- **상품 후기 메이커**: 상품 상세 이미지 여러 장(업로드/Ctrl+V 누적 지원) 및 정보 다중 추출, 사용자 메모 기반으로 후기형 블로그 글 초안 자동 생성 (도입부~구매 전 확인점 구조화, 내부 라벨 노출 차단, 건강/효능 과장 표현 금지)
- **AI 주제/제목 추천**: 사용자 키워드 기반 트렌디한 주제 및 클릭을 부르는 제목 후보 제안. 기본 제목 후보 3종(검색형, 후기형, 클릭형)을 우선 노출하고 비교형, 선택형 등 추가 후보는 "제목 더 보기" 버튼으로 펼쳐 확인 가능하게 UI를 구현합니다.
- **블로그 원고 결과 (14종 원고 패키지)**: 메인/보조 키워드, 검색 의도 분석, 홈피드 클릭 포인트, 제목 후보 5개, 최종 추천 제목, 첫 문장 후보, 블로그 본문, 사진 배치 가이드, 업체/상품 정보 정리, 이런 분께 추천해요, FAQ, 해시태그, 최종 검수표로 이루어진 구조화된 JSON 응답 스키마와 결과 UI 컴포넌트. 화면에서는 **선택 제목 바로 아래에 본문 원고를 즉시 노출**하고, 해시태그, 사진 배치 가이드, 추천 대상, FAQ, 글 발행 전 확인사항 순으로 배치합니다. 메인/보조 키워드 및 검색 의도 분석 데이터는 하단 접힘 영역(**'자세한 글 분석 보기'** 또는 **'글 작성 참고 정보'**)으로 이동하여 렌더링됩니다.
- **소제목 출력 및 복사 필터**: 본문 에디터 및 결과 화면(HTML 뷰)에서는 소제목 텍스트가 CSS 효과에 의해 굵게 보이지만, `복사` 버튼을 눌러 클립보드에 담을 때는 소제목 앞뒤의 `*`, `**`, `###` 같은 마크다운 특수기호가 완전 필터링(삭제)됩니다. 줄바꿈 개행은 정상적으로 보존되어 네이버 스마트에디터에 복사-붙여넣기 시 불필요한 마크다운 문자를 수동으로 지우지 않아도 되도록 최적화합니다.
- **카테고리별 필수 확인 기준 검증**: 맛집, 카페, 아이와 갈만한 곳, 상품 리뷰, 생활용품/계절템, 체험단/협찬 리뷰별 필수 항목(주차, 가격, 영업시간, 웨이팅, 사용 기간, 제품 효과 등)의 기입 여부를 생성 엔진 및 프론트 검수에서 실시간으로 대조합니다. 정보가 누락된 경우 AI 임의 조작을 차단하고 `[확인 필요]` fallback 문구를 삽입합니다. 협찬 여부 미기입 시 최종 검수표에 `[협찬 여부 확인 필요]` 경고를 강제 렌더링합니다.
- **글 발행 전 확인사항 (블로그 글 구조 점검)**: 상위노출 보장 표현을 배제한 운영 보조 시스템. 제목 키워드 반영, 첫 문단 키워드 밀도, 보조 키워드 분산, 제목/본문/검색 의도 일치, 경험/체크포인트 포함, 금칙어 및 과장 표현 여부를 검수하며, 화면에는 쉬운 명칭인 **"글 발행 전 확인사항"** 패널로 노출합니다.
- **작성 프로필 (브랜드/상품 프로필) 관리**: 단일 기본값 구조에서 벗어나, 품목/브랜드별로 사용자 유형, 업종/주제, 글 목적, 말투, 금지어, CTA 톤, 목표 글자수, 브랜드명/매장명, 지역, 핵심 강점, 강조 포인트, 주소, 영업시간, 가격/가격대, 구매처, 문의 방법, 배송/교환 정보를 여러 개 다중 저장하고 불러오기
- **이미지 삽입 가이드**: 본문 안에 `[여기에 이미지를 넣어주세요 이미지 N]` 표시를 직접 넣고, 하단 이미지 추천 카드와 번호를 연결. 추천 카드에는 이미지 컨셉, 직접 촬영 추천(블로그용 우선), AI 이미지 프롬프트(티스토리용), 사이트 검색어 및 복사 버튼을 모두 제공
- **스마트 해시태그**: 지역명+카테고리, 카테고리+후기, 상품/브랜드, 검색 의도형 롱테일, 구매/방문/비교 키워드를 포함한 10~15개 해시태그 패키지를 구성하며, 네이버 블로그에 바로 붙여넣을 수 있는 전체/그룹별 복사 기능 지원
- **판매용 접속 제어**: 크몽/직접판매 URL 유출 차단을 위해 접속코드 입력(예: `MGO-TEST7`, `MGO-BETA30`)을 통한 기간제 사용권 인증. 브라우저 localStorage로 인증을 유지하고 만료 시 즉시 접속 제한 (※ 정식 판매 전환 시 서버 검증으로 업그레이드 권장)
- **기본 대시보드**: 최근 작업 내역 확인 및 작업 재개 기능
- **댓글 응답 관리 (베타 포함)**: 캡처 이미지 및 단순화된 수동 입력(작성자명 선택화) 기반 댓글 추출, 템플릿을 탈피한 감정/의도 맞춤형 상호대댓글 생성 (URL 자동화는 고급 브리지 기능으로 분리)

## 2. 메뉴 구조 및 핵심 기능 (1줄 설명)

| 메뉴명 | 핵심 기능 (1줄 설명) | 비고 |
| :--- | :--- | :--- |
| **대시보드** | 최근 작성한 콘텐츠 내역과 운영 현황을 한눈에 확인 | 관리의 시작점 |

| **상품 후기 메이커** | 다중 상품 이미지/정보 추출과 메모만으로 후기형 자동 생성 | 리뷰 특화 작업실 |

## 3. 사용자 흐름 및 전략 (Managed/Semi-Auto)

1. **아이디어 선택 및 직접 수정**: 추천 주제(글 방향) 후보를 사용자가 직접 수정하거나 그대로 선택할 수 있는 관리형 인터페이스 제공
2. **초안 피드백 및 글 순서 정하기**: AI가 제안한 제목 후보 3종(검색/후기/클릭형)을 우선 보고 추가 후보는 '제목 더 보기'로 펼쳐 확인하며, 소제목들을 '글 순서 정하기'로 편집하고 개별 메모("이 부분에 꼭 넣고 싶은 내용을 적어주세요. 안 적어도 괜찮아요. (선택)")를 작성하도록 유도
3. **최종 완성**: 사용자의 확인을 거쳐 '이 구성으로 본문 만들기'로 본문 작성을 완료하는 '반자동 생산' 프로세스 (생성 완료 시 자연스럽게 결과 확인 영역으로 이동)
4. **검수형 UX 지향**: 생성 후 '글 발행 전 확인사항' (블로그 글 구조 점검) 패널에서 제목, 첫 문단, 소제목, 키워드가 검색에 잘 맞는지 최종 확인
5. **상품형 UX 지향**: 콘텐츠 메이커는 처음 사용 가이드(PC 카드형, 모바일 접힘형)와 예시 자동 채우기를 제공하고, 모바일 우선 가독성 확보, '내 보관함 저장' 등 친화적 용어 사용, 결과 화면 렌더링 순서 최적화(제목 하단 본문 즉시 노출, 그 외 결과 순차 정렬, 분석 데이터는 '자세한 글 분석 보기' 또는 '글 작성 참고 정보' 하단 접힘 처리), 직관적인 이미지 삽입 위치 연결 및 3분할 복사 옵션, 네이버 블로그 연동을 위한 전체/그룹별 해시태그 복사 제공

## 4. 초기 폴더 구조 제안

기능별 모듈화와 확장성을 고려한 구조입니다.

```text
/a-blog-allinone
  ├── /frontend          # React + Vite + Tailwind (프리미엄 UI)
  │   ├── /src
  │   │   ├── /components # 재사용 가능한 UI 컴포넌트
  │   │   ├── /pages      # 각 메뉴별 페이지
  │   │   ├── /store      # 상태 관리 (아이디어, 초안 등)
  │   │   └── /styles     # 전역 CSS 및 디자인 토큰
  ├── /backend           # Node.js + Express (로직 처리)
  │   ├── /api           # API 엔드포인트
  │   ├── /services      # AI(Gemini), 데이터 처리 로직
  │   └── /modules       # 외부 연동 모듈 (댓글 자동화 등)
  └── /shared            # 프론트/백 공용 타입 정의 (TS)
```

## 5. 모듈 분리 및 연결 전략 (Legacy Integration)

기존 `blog-automation`의 기능을 직접 가져오는 대신, **'브릿지(Bridge) 패턴'**을 사용합니다.
- 정적 베타 URL 환경에서는 URL 자동 댓글 수집/자동 등록 기능을 제외하고, 이를 로컬 `blog-automation` 브리지 기능(정식 상품화 단계의 고급 기능)으로 분리하여 역할을 명확히 나눕니다.
- 정적 베타 모드에서도 콘텐츠 메이커, 블로그 글 구조 점검, 접속코드 인증, 수동 댓글 입력, 캡처 업로드, 대댓글 생성/복사 기능은 유지합니다. URL 자동 수집/자동 등록만 로컬 브리지 기반 고급 기능으로 관리합니다.
- 접속코드 검증은 `frontend/src/lib/accessControl.js`에 분리하여 향후 Supabase 또는 Cloudflare Workers/KV 기반 서버 검증으로 교체합니다.

## 6. 1차 베타 피드백 반영 현황 (구현 완료)

이번 구현 단계를 통해 사용성 및 생성 퀄리티에 대한 핵심 피드백이 전면 반영되었습니다.

1. **처음 사용 가이드 및 예시 채우기**: 콘텐츠 메이커에 초보자를 위한 처음 사용 가이드(PC 카드형, 모바일 접힘형)와 예시 자동 채우기 기능 추가
2. **작성 프로필 (브랜드/상품 프로필) 다중 저장**: 품목/브랜드별 작성 프로필을 여러 개 저장하고 목록에서 선택 불러오기
3. **본문 내 이미지 삽입 위치 표시**: 사용자가 삽입 포인트를 쉽게 알 수 있도록 본문에 `[여기에 이미지를 넣어주세요 이미지 N]` 직접 삽입
4. **이미지 추천 연동**: 하단 이미지 추천 카드가 본문 삽입 번호 위치와 정확히 연결되고 실무 중심으로 이미지 컨셉, 직접 촬영 추천, AI 프롬프트, 검색어 및 복사 버튼 제공
5. **복사 옵션 개선**: 본문만 복사 / 본문+이미지 표시 포함 복사 / 이미지 검색어만 복사, 네이버 블로그 붙여넣기에 최적화된 전체/그룹별 해시태그 복사 기능 제공. 특히 소제목 마크다운 기호(`*`, `**`, `###`)를 제거하는 복사 필터 추가.
6. **UX 및 예시 문구 개선**: 특정 브랜드 예시를 범용 상품군(수분크림, 미니청소기 등)으로 대체. 상품 후기 메이커 첫 화면을 단순화하고 이미지 등록 전에는 추출 영역을 접어("이미지를 넣으면 확인 가능" 안내만 표시) 모바일 생성 동선 최적화 (자동 펼침 및 OCR 원문 분리 지원)
7. **블로그 글 구조 점검 강화**: "검색 의도"라는 어려운 용어 대신, 결과 영역에 "검색에 잘 맞는 글인지 확인"하는 블로그 글 구조 점검 패널 추가
8. **답변형 FAQ 구조**: 첫 문단 핵심 답변, 소제목별 결론형 첫 문장, 조건부 FAQ 3개 생성
9. **접속코드 게이트**: 정적 베타/1차 판매용 접속코드와 만료일 기반 간단 인증 추가
10. **댓글 응답 품질 고도화**: 기계적인 템플릿(예: "문의하신 부분은")을 금지하고 원댓글의 핵심 표현(하얀가루, 가루날림 등)과 감정을 살리는 상호대댓글로 개선
11. **수동 댓글 UX 간소화**: 작성자명을 선택값으로 변경하여 댓글 내용만 붙여넣어도 바로 대댓글 생성이 가능하도록 개선
12. **다시 생성(Variation) 보장**: 생성된 기존 문장과 다른 문장으로 재생성되도록 알고리즘 개선
13. **정적 베타 모드 지원**: 정적 웹 환경에서도 콘텐츠 메이커 전체와 수동/캡처 기반 대댓글 생성/복사 기능이 문제없이 구동됨
14. **자동화 브리지 분리**: URL 자동 수집/자동 등록 기능은 정식 확장팩인 로컬 `blog-automation` 브리지 기반으로 별도 관리
15. **글 순서 정하기 및 메모 연동 (신규)**: 생성된 개요 소제목들을 직접 수정/추가("항목 추가")/삭제할 수 있고, PC/모바일 대응용 Drag & Drop 및 위/아래 이동 버튼("글 순서 정하기")을 지원하며, 소제목별 메모 입력 필드 안내 문구를 "이 부분에 꼭 넣고 싶은 내용을 적어주세요. 안 적어도 괜찮아요. (선택)"로 개선하여 선택 입력임을 명시
16. **5대 주제별 구조 차별화 (신규)**: 정보형/후기형/비교형/방문형/상품형의 템플릿 구조를 확실히 구별하여 같은 키워드라도 도입부, 소제목, 결론부 구조를 독창적으로 생성.
17. **구어체 말투 강제 (신규)**: 친근한 말투 선택 시 '~했어요, ~더라고요, ~느껴졌어요, ~같아요' 어미를 활용하고 딱딱한 투의 표현 지양.
18. **블로그 원고 결과 14종 통합 패키지 (신규)**: 메인/보조 키워드, 의도 분석, 후킹 포인트, 제목 후보 5선, 추천 제목, 도입 문장, 본문, 사진 배치 가이드, 정보 정리, 추천 타겟, FAQ, 해시태그, 최종 검수표 데이터 구조화.
19. **카테고리별 필수 기입 및 검증 (신규)**: 맛집, 카페 등 6대 카테고리 필수 데이터 누락 시 `[확인 필요]`, 협찬 여부 누락 시 `[협찬 여부 확인 필요]` 경고 표시 및 비경험 글의 정보 정리형/방문 전 참고형 톤앤매너 전환.
20. **추가 UX 피드백 전면 반영 (신규)**:
    - **글 방향 직접 수정**: 추천 주제(글 방향) 후보를 사용자가 직접 텍스트 인풋에서 수정 가능하게 하고 이를 제목/글 순서 정하기/본문 생성 프롬프트에 반영.
    - **결과 레이아웃 순서 재배치**: 선택한 제목 아래에 본문 원고를 즉시 배치하고, 해시태그, 이미지 가이드, 추천 대상, FAQ, 글 발행 전 확인사항 순서로 배치하며 키워드/의도 분석 정보는 하단 접힘('자세한 글 분석 보기' 또는 '글 작성 참고 정보')으로 격리.
    - **제목 후보 UI 최적화**: 기본 제목 후보 3종(검색형/후기형/클릭형) 우선 노출 및 추가 제목은 '제목 더 보기' 펼침으로 구성.
    - **버튼 및 라벨 쉬운 용어 개편**: "글 순서 정하기", "글 구성 만들기", "항목 추가", "이 구성으로 본문 만들기"로 변경 완료.

## 7. 1차 개발 우선순위 및 순서

1. **Step 1 (환경 구축)**: Vite + React + Tailwind + Express 프로젝트 기본 구조 세팅
2. **Step 2 (콘텐츠 메이커 UI 및 글 순서 정하기)**: '주제 생성 및 글 방향 수정 -> 제목 생성(기본 3종 우선 노출) -> 글 구성 만들기 -> 글 순서 정하기 UI(Drag & Drop, 위/아래 이동 버튼, 소제목별 꼭 넣고 싶은 내용 메모 박스) -> 이 구성으로 본문 만들기' 흐름의 UI 및 컴포넌트 구현
3. **Step 3 (AI 엔진 연동 및 5대 주제/말투 고도화)**: Gemini API를 연결하여 수정한 글 방향, 글 순서, 소제목별 메모가 실시간 반영되고 5대 주제별 구조가 명확하며 친근한 구어체 기반의 블로그 원고 결과 데이터 생성 로직 탑재
4. **Step 4 (카테고리 검증 및 렌더링/복사 필터)**: 카테고리별 필수 항목 검사 및 결과 화면 순서 재배치(제목 아래 본문 즉시 노출, 그 아래 해시태그/가이드/FAQ/확인사항 순 정렬, 분석 정보 하단 접힘) 구현 및 클립보드 복사 시 마크다운 기호 정규식 제거 필터 개발
5. **Step 5 (대시보드/보관함/설정)**: 로컬 스토리지 또는 DB를 활용한 초안 저장 기능, 대시보드 리스트 및 로컬 브리지 연결 포인트 확보
