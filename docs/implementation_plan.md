# [계획서] 블로그 올인원 운영 시스템 (Blog All-in-One System)

본 프로젝트는 블로거와 소상공인을 위한 통합 운영 솔루션으로, AI 기반의 콘텐츠 생산과 관리 자동화를 목표로 합니다. 특히 크몽 판매용 상품으로서 '프리미엄한 디자인'과 '사용자 편의성'을 최우선으로 합니다.

## User Review Required

> [!IMPORTANT]
> **MVP 범위 확인**: 초기 단계에서 모든 자동화 기능을 넣기보다, 핵심 가치인 '글쓰기 생산성'에 집중할 예정입니다. 아래 제안된 MVP 범위가 판매 전략과 일치하는지 확인 부탁드립니다.

> [!NOTE]
> **디자인 테마**: '안티그래비티'의 정체성을 담아 다크 모드 기반의 세련된 글래스모피즘(Glassmorphism) 스타일을 제안합니다.

## 1. MVP 기능 범위 (1차 고정)

사용자가 '직접 관리하는 재미'를 느끼면서도 '반자동'으로 생산성을 높일 수 있는 핵심 기능에 집중합니다.

- **AI 주제 추천**: 키워드 기반으로 사용자가 고를 수 있는 5가지 포스팅 방향 제안
- **제목 & 본문 초안 작성**: 서론-본론-결론 구조의 고퀄리티 초안 생성 (초보자용 키워드/예시 가이드 적용 및 직관적인 이미지 삽입 위치 번호 표시)
- **SEO/AEO 운영 보조**: 상위노출 보장 등 허위 표현을 철저히 금지하며, 네이버 블로그 글쓰기 흐름에 맞춘 '검색 의도 기반 보조 시스템'으로 동작합니다. 메인/보조 키워드 자연 반영, 금지어 체크, 해시태그 그룹화, 목표 글자수에 맞는 소제목 수 분배, 마무리 CTA 적용 등을 통합 검수합니다.
- **AEO형 답변 구조**: 검색자가 원하는 답을 빠르게 찾도록 첫 문단 핵심 답변 배치, 질문형 소제목, 소제목별 결론 우선 문장을 적용합니다. 또한 누구에게 필요한지/선택 기준/확인 사항 등의 정보를 포함하며 조건부 FAQ(2~3개)를 구성합니다.
- **이미지 삽입 가이드**: 본문 안에 `[이미지 삽입 추천 N]` 표시를 직접 넣고, 하단 이미지 추천 카드와 번호를 연결. 목표 글자수별 추천 수는 1200자 2개, 1500자 3개, 2000자 이상 4개로 조정
- **스마트 해시태그**: 검색 의도를 반영하여 논리적으로 그룹화된 최적 해시태그 패키지 구성
- **판매용 접속 제어**: 크몽/직접판매 URL 유출 차단을 위해 접속코드 입력(예: `MGO-TEST7`, `MGO-BETA30`)을 통한 기간제 사용권 인증. 브라우저 localStorage로 인증을 유지하고 만료 시 즉시 접속 제한 (※ 정식 판매 전환 시 서버 검증으로 업그레이드 권장)
- **기본 대시보드**: 최근 작업 내역 확인 및 작업 재개 기능
- **댓글 응답 관리 (베타 포함)**: 캡처 이미지 및 단순화된 수동 입력(작성자명 선택화) 기반 댓글 추출, 템플릿을 탈피한 감정/의도 맞춤형 상호대댓글 생성 (URL 자동화는 고급 브리지 기능으로 분리)

## 2. 메뉴 구조 및 핵심 기능 (1줄 설명)

| 메뉴명 | 핵심 기능 (1줄 설명) | 비고 |
| :--- | :--- | :--- |
| **대시보드** | 최근 작성한 콘텐츠 내역과 운영 현황을 한눈에 확인 | 관리의 시작점 |
| **콘텐츠 메이커** | 주제 선정부터 초안, 해시태그까지 원스톱으로 생성 | 메인 작업실 |
| **내 보관함** | 작성된 모든 초안을 날짜별/주제별로 안전하게 보관 | 데이터 자산화 |
| **고급 설정** | API 키 관리 및 (추후) 댓글 자동화 브릿지 연결 | 확장성 공간 |

## 3. 사용자 흐름 및 전략 (Managed/Semi-Auto)

1. **아이디어 선택**: 완전 자동이 아닌, 사용자가 AI 추천 리스트 중 하나를 직접 선택 (관리형)
2. **초안 피드백**: AI가 쓴 글을 바로 복사하는 것이 아니라, 에디터에서 사용자가 터치할 수 있게 유도
3. **최종 완성**: 사용자의 확인을 거쳐 발행 준비를 완료하는 '반자동 생산' 프로세스 (생성 완료 시 자연스럽게 결과 확인 영역으로 이동)
4. **검수형 UX 지향**: 생성 후 SEO/AEO 점검 패널에서 검색 의도 기반 구조, 키워드 자연 반영, 답변형 문장 포함, 본문 흐름 점검을 확인
5. **상품형 UX 지향**: 콘텐츠 메이커는 PC 카드형/모바일 접힘형 처음 사용 가이드와 예시 자동 채우기를 제공하고, 모바일 우선 가독성 확보, '내 보관함 저장' 등 친화적 용어 사용, 필수 입력 최소화(예: 댓글 작성자명 선택화), 직관적인 이미지 삽입 위치 연결 및 3분할 복사 옵션 제공

## 4. 초기 폴더 구조 제안

기능별 모듈화와 확장성을 고려한 구조입니다.

```text
/a-blog-allinone
  ├── /frontend          # React + Vite + Tailwind (프리미엄 UI)
  │   ├── /src
  │   │   ├── /components # 재사용 가능한 UI 컴포넌트
  │   │   ├── /pages      # 각 메뉴별 페이지
  │   │   ├── /store      # 상태 관리 (아이디어, 초안 등)
  │   │   └── /styles     # 전역 CSS 및 디자인 토큰
  ├── /backend           # Node.js + Express (로직 처리)
  │   ├── /api           # API 엔드포인트
  │   ├── /services      # AI(Gemini), 데이터 처리 로직
  │   └── /modules       # 외부 연동 모듈 (댓글 자동화 등)
  └── /shared            # 프론트/백 공용 타입 정의 (TS)
```

## 5. 모듈 분리 및 연결 전략 (Legacy Integration)

기존 `blog-automation`의 기능을 직접 가져오는 대신, **'브릿지(Bridge) 패턴'**을 사용합니다.
- 정적 베타 URL 환경에서는 URL 자동 댓글 수집/자동 등록 기능을 제외하고, 이를 로컬 `blog-automation` 브리지 기능(정식 상품화 단계의 고급 기능)으로 분리하여 역할을 명확히 나눕니다.
- 정적 베타 모드에서도 콘텐츠 메이커, SEO/AEO 점검, 접속코드 인증, 수동 댓글 입력, 캡처 업로드, 대댓글 생성/복사 기능은 유지합니다. URL 자동 수집/자동 등록만 로컬 브리지 기반 고급 기능으로 관리합니다.
- 접속코드 검증은 `frontend/src/lib/accessControl.js`에 분리하여 향후 Supabase 또는 Cloudflare Workers/KV 기반 서버 검증으로 교체합니다.

## 6. 1차 베타 피드백 반영 현황 (구현 완료)

이번 구현 단계를 통해 사용성 및 생성 퀄리티에 대한 핵심 피드백이 전면 반영되었습니다.

1. **처음 사용 가이드 및 예시 채우기**: 콘텐츠 메이커에 초보자를 위한 처음 사용 가이드(PC 카드형, 모바일 접힘형)와 예시 자동 채우기 기능 추가
2. **본문 내 이미지 삽입 위치 표시**: 사용자가 삽입 포인트를 쉽게 알 수 있도록 본문에 `[이미지 삽입 추천 N]` 직접 삽입
3. **이미지 추천 연동**: 하단 이미지 추천 카드가 본문의 삽입 번호 위치와 정확히 연결됨
4. **복사 옵션 3분할**: 본문만 복사 / 본문+이미지 표시 포함 복사 / 이미지 키워드만 복사 기능 제공
5. **SEO/AEO 점검 강화**: 결과 영역에 검색 의도 기반 구조, 키워드 자연 반영, 답변형 문장 구조, 본문 흐름 점검 패널 추가
6. **AEO형 FAQ 구조**: 첫 문단 핵심 답변, 소제목별 결론형 첫 문장, 조건부 FAQ 2~3개 생성
7. **접속코드 게이트**: 정적 베타/1차 판매용 접속코드와 만료일 기반 간단 인증 추가
8. **댓글 응답 품질 고도화**: 기계적인 템플릿(예: "문의하신 부분은")을 금지하고 원댓글의 핵심 표현(하얀가루, 가루날림 등)과 감정을 살리는 상호대댓글로 개선
9. **수동 댓글 UX 간소화**: 작성자명을 선택값으로 변경하여 댓글 내용만 붙여넣어도 바로 대댓글 생성이 가능하도록 개선
10. **다시 생성(Variation) 보장**: 생성된 기존 문장과 다른 문장으로 재생성되도록 알고리즘 개선
11. **정적 베타 모드 지원**: 정적 웹 환경에서도 콘텐츠 메이커 전체와 수동/캡처 기반 대댓글 생성/복사 기능이 문제없이 구동됨
12. **자동화 브리지 분리**: URL 자동 수집/자동 등록 기능은 정식 확장팩인 로컬 `blog-automation` 브리지 기반으로 별도 관리

## 7. 1차 개발 우선순위 및 순서

1. **Step 1 (환경 구축)**: Vite + React + Tailwind + Express 프로젝트 기본 구조 세팅
2. **Step 2 (콘텐츠 메이커 UI)**: 사용자가 가장 먼저 마주할 '주제 입력 -> 결과 확인' 흐름의 UI 구현
3. **Step 3 (AI 엔진 연동)**: Gemini API를 연결하여 실제 주제 및 본문 생성 로직 탑재
4. **Step 4 (대시보드/보관함)**: 로컬 스토리지 또는 DB를 활용한 초안 저장 기능 및 대시보드 리스트 구현
5. **Step 5 (고급 설정/브릿지)**: 확장성을 위한 설정 페이지 및 기존 모듈 연결 포인트 확보
